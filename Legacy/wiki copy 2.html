Kernel Documentation

The Meta-Mind architecture separates logic (Kernel) from presentation (Shell). This document outlines the core protocols driving the Federated Mapstate.

1. The Kernel Pattern

Meta-Mind uses a singleton class MetaMindKernel (instantiated as MM) to manage the application state. The UI layer subscribes to state changes via MM.subscribe(), ensuring purely reactive rendering.

// meta-mind-core.js class MetaMindKernel { constructor() { this.state = { nodes: [], edges: [], viewport: {...} }; this.listeners = []; } notify() { this.listeners.forEach(cb => cb(this.state)); } }

2. Federated Data Schema (v3.0)

Nodes are the atomic units of the metaverse. In v3.0, they include origin and policy objects to support future federation and permissions.

Node Object

{ "id": "uuid-v4", "type": "concept", // widget, section, logic "title": "Core System", "content": "...", "position": { "x": 0, "y": 0 }, "origin": { "creator": "user_hash", "created": "ISO-8601" }, "policy": { "access": "public" } }

Edge Object

{ "id": "uuid-v4", "source": "node-id-a", "target": "node-id-b", "type": "child", // parent, sibling, logic "meta": { "weight": 1.0, "context": "local" } }

3. Core API Reference

MM.addNode(config) Returns: String (UUID)

Creates a new node and adds it to the state. Automatically handles origin metadata.

MM.addEdge(src, tgt, type) Returns: Void

Links two nodes. Prevents duplicate edges automatically. Triggers notify().

MM.autoLayout() Returns: Void

Executes the hierarchical layout algorithm. It identifies root nodes (no parents) and recursively positions children downwards, calculating branch width to prevent overlap.
